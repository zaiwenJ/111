# Apple-Inspired 设计深度调优计划 (Deep Refinement Plan)

## Summary (总结)
在完成初步的类苹果（Apple-inspired）风格重构后，页面已经具备了高级的质感与毛玻璃效果。为了将极简主义推向极致，达到真实的 iOS/macOS 级别的高级感，我们需要进行第二阶段的“去噪与精雕”。本计划详细列出了接下来即将进行的深度调优步骤。

## Current State Analysis (当前状态分析)
目前的 HTML 页面（`差距分析预览页面.html`）已经引入了 `SF Pro` 字体、大圆角（`rounded-2xl`）以及 `backdrop-blur` 模糊质感。但距离纯正的苹果风格，仍有一些可以优化的“传统 SaaS 残留”：
1. **显性边框过多**：卡片和矩阵容器仍有灰色的实线边框（`border-gray-100`），这在苹果设计中往往被更微妙的阴影或背景色差替代。
2. **色彩饱和度偏高**：部分标签（Tags）如绿色、紫色、橙色的底色饱和度仍然偏高。苹果倾向于使用极度克制的低饱和灰色底衬，配合高饱和的小色块点缀（或直接使用 `#0071e3` Apple Blue）。
3. **动效曲线不够原生**：虽然有过渡动画，但缺少 iOS 原生的“弹簧阻尼感（Spring Animation）”。
4. **字体层级对比**：文字颜色的黑度还可以进一步规范（例如采用苹果经典的 `#1d1d1f` 作为主色，`#86868b` 作为次级说明色）。

## Proposed Changes (具体修改方案)

### 1. 极致“去线化”与阴影重塑 (De-bordering & Shadow Tuning)
- **What**: 移除或极度弱化左侧卡片（Card）、右侧申报矩阵（Matrix）的物理边框（`border`）。
- **How**: 
  - 删除多余的 `border border-gray-100/80`。
  - 为卡片引入苹果原生的多层扩散阴影（例如：`shadow-[0_2px_10px_rgba(0,0,0,0.04),_0_10px_30px_rgba(0,0,0,0.02)]`）。
  - 背景色调优，卡片使用纯白 `#ffffff`，底色使用更深一点的极简灰 `#f5f5f7` 以形成自然对比。

### 2. 色彩降噪与苹果标准色盘 (Color Denoising & Apple Palette)
- **What**: 统一并克制页面的全局强调色，重构标签（Pill）的色彩系统。
- **How**:
  - 全局的主题强调色（如目前的 Indigo）全部替换为 **Apple Blue (`#0071e3`)** 或类似的亮蓝色。
  - 成果卡片内的所有彩色背景标签（Tag）全部降噪为**统一的浅灰色底 (`bg-gray-100`) + 黑色/深灰色文字**，仅保留极小面积的彩色指示点（Dot）或小图标来区分类型，消除视觉杂乱感。

### 3. 苹果原生排版微调 (Typography & Hierarchy)
- **What**: 强化标题的紧凑感，规范正文与次要文本的经典苹果灰。
- **How**:
  - 核心标题（如“正教授职称申报”、“A1 教学核心业绩”）增加更强的字重（`font-semibold` 到 `font-bold`）并配合更紧凑的字距（`tracking-tight` 到 `tracking-tighter`）。
  - 全局主文本颜色使用 `#1d1d1f`（更深邃沉稳的黑）。
  - 次级辅助说明文字统一使用 `#86868b`。

### 4. 引入弹簧阻尼动效 (Spring Animations)
- **What**: 替换 CSS 原生的 `ease` 或 `ease-out`，使用模拟物理弹簧的贝塞尔曲线。
- **How**: 
  - 为卡片拖拽、Hover 浮起、以及底部 Smart Dock 的展开，应用 `cubic-bezier(0.32, 0.72, 0, 1)`（类似 iOS 的平滑弹性过渡）。
  - 拖拽时的卡片缩放比例微调为 `scale(0.98)`（更微妙），增加深度的投射阴影。

### 5. 毛玻璃层级细化 (Refined Vibrancy)
- **What**: 让 Sticky Header 和 侧边栏的玻璃质感更纯净。
- **How**: 
  - 提高 `backdrop-blur` 的像素值（如 `backdrop-blur-2xl`），同时将背景色的透明度调整为更通透的白（如 `bg-white/60`），确保下方文字滑过时有极佳的磨砂折射感。

## Assumptions & Decisions (假设与决策)
- **决策**：我们假设您希望牺牲掉一部分“五颜六色”的传统 SaaS 标签分类法，换取苹果极其克制、高级、以黑白灰为主+单色点缀的极简视觉体验。
- **决策**：完全去除了显性边框，意味着需要依赖阴影和留白来区分层级，因此卡片的内部间距（Padding）可能会略微增大，让页面更有“呼吸感”。

## Verification steps (验证步骤)
1. 使用浏览器打开修改后的 `差距分析预览页面.html`。
2. 验证所有边框是否被更柔和的阴影替代。
3. 拖拽卡片，感受弹簧动效和颜色的克制感。
4. 查看底部面板展开时的丝滑度与颜色层级。
